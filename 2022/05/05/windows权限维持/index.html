自启动注册表植入后门（开机自启）

windows提供了开机自启动注册表，每次开机完成后，计算机会自动遍历自启动注册表下的键值。
2个常用的开机自启注册表键值

1.HKEY_CURRENT_USER\Software\Mircosoft\Windows\CurrentVersion\Run
在该注册表下添加键test，值poc.exe

2.HKEY_LOCAL_MACHINE\Software\Microsoft\Windows\CurrentVersion\Run
在该注册表下添加键test，值poc.exe

PS:HKEY_CURRENT_USER代表当前用户，需要该用户登录才能触发后门自启
HKEY_LOCAL_MACHINE代表所有用户，所有用户登录都可实现自启

映像劫持

用户打开某应用程序时，操作系统会给外壳程序（例如“explorer.exe”）发布相应的指令，其中包含有执行程序的路径和文件名，然后由外壳程序来执行该程序。在该过程中，Windows还会在注册表的上述路径中查询所有的映像劫持子键，如果存在和该程序名称完全相同的子键，就查询对应子健中包含的“dubugger”键值名，并用其指定的程序路径来代替原始的程序，之后执行的是遭到“劫持”的虚假程序。

劫持对应的程序后，打开该程序运行的是我们指定（替换）的程序

玩法1：直接劫持，原程序无法运行

  1. 在注册表路径下创建一个项，项名为要劫持的exe名称，可以为任意一个可以运行的exe程序。并在该项创建一个Debugger的键值（名字只能为Debugger），键值填运行的恶意exe。

对notepad++程序进行劫持

此时运行notepad，实际执行的是poc.exe,notepad不会被执行。

玩法2：原程序退出后，自动执行恶意程序
此时打开notepad再关闭，就会弹出计算机

logon scripts 后门

Windows 登录脚本，当用户登录时触发，Logon Scripts 能够优先于杀毒软件执行，绕过杀毒软件对敏感操作的拦截。
用户一登录成功执行poc.exe
